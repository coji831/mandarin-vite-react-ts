# Backend Working Rules

- Preserve layer boundaries across routes, controllers, services, repositories, and schema work.
- Treat API contract, auth, validation, and schema changes as high-signal changes that require focused verification.
- Keep rollout risk visible: migrations, environment assumptions, and external dependencies must be surfaced early.
- Route security-sensitive flows through the Security Auditor before closure.
- Record contract changes, blockers, and verification gaps in `.ai_ledger.md`.
