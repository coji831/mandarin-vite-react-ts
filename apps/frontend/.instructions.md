# Frontend Working Rules

- Follow the existing feature-folder structure under `apps/frontend/src/features/`.
- Keep state changes local and explicit: reducer, action, selector, and test updates should stay aligned.
- Prefer focused UI changes over broad visual rewrites unless the task explicitly calls for them.
- Route contract-sensitive work back through the orchestrator if backend assumptions change.
- Record integration assumptions, review findings, and verification gaps in `.ai_ledger.md`.
