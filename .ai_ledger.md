## Completion Notes

When all verification checks pass and all work is complete:

1. Document the evidence (root cause, fix, test results, auditor sign-off)
2. Update the "Completion Promise" field **in the Current Objective section above** from `pending` to the appropriate promise tag
3. The Stop hook will detect the non-pending value and allow the chat to exit

**Critical:** The promise tag must be written to the `Completion Promise:` field in **Current Objective** during your work turn — not at exit time. The Stop hook fires when you try to stop, so the ledger must already reflect the promise by then.

Supported promises:

- `<promise>WORK_PACKAGE_COMPLETE</promise>` — All work done, all verifications pass, ready to close
- `<promise>WORK_PACKAGE_BLOCKED</promise>` — Work blocked by external dependency, blocker documented in Blockers section
- `<promise>ESCALATION_REQUIRED</promise>` — Issue exceeds agent scope, escalated to human decision
